# D-Tag: Tag ≠ Segment

**Date**: 2026-04-15
**Source**: Session กับ Tee — Hyundai Card D-Tag Framework

## Lesson

การออกแบบ customer intelligence ที่มีพลังต้องเปลี่ยน paradigm จาก **"ลูกค้าอยู่ในกลุ่มไหน"** → **"ลูกค้ามี tag อะไรบ้าง"**

Tag ที่ดีมี 3 คุณสมบัติ:
1. **Multi-dimensional** — ลูกค้า 1 คนมีได้หลาย tag พร้อมกัน
2. **Dynamic** — refresh รายเดือนตาม behavior จริง ไม่ static
3. **Confidence-scored** — ทุก tag มี score 0–1 แทน binary assign

## Why It Matters

Traditional segmentation บังคับให้ลูกค้าอยู่ใน 1 กลุ่ม ทำให้ campaign targeting และ product design ไม่ตรงจริง Tag approach ให้ความยืดหยุ่นในการ combine dimensions ตาม use case

## How to Apply

- เริ่มจาก data ที่มีอยู่แล้ว (MCC + Merchant Name) ก่อนหา data ใหม่
- Phase 1 ควร prove value ด้วย tag ที่ build จาก existing data
- Define threshold แยกตาม use case ไม่ใช่ global threshold เดียว
