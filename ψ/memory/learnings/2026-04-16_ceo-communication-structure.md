# CEO Communication Structure

**Date**: 2026-04-16
**Source**: Session กับ Tee — Customer Tagging CEO Presentation & Memo

## Lesson

การสื่อสารกับ CEO หรือ executive ต้องเรียงข้อมูลตาม mental model ของเขา ไม่ใช่ตาม logic ของ solution

## Pattern: Big Picture First

```
❌ Solution-first (engineer mindset)
Problem → What we built → How it works → Results

✅ Vision-first (CEO mindset)
Where we're going (big picture) → Where we are now → What we need
```

ใน session นี้: เปิดด้วย UNIVERSE Platform 5 Layers ก่อน แล้วค่อยบอกว่า "ตอนนี้เราอยู่ Layer 1" ทำให้ทุกอย่าง make sense ทันที เหมือน "you are here" บนแผนที่ที่ต้องเห็นแผนที่ก่อน

## Pattern: Same Content, Multiple Formats

Content เดียวกันถูกแปลงเป็น 5 formats ใน session เดียว:
- Engineer spec (technical detail)
- Engineer presentation (structured visual)
- CEO presentation (narrative visual)
- CEO PowerPoint (designed deck)
- CEO memo (prose for reading)

แต่ละ format มี logic การเรียงข้อมูลต่างกัน — อย่า assume ว่า copy-paste จาก format หนึ่งไปอีก format จะทำงาน

## How to Apply

ก่อน draft communication artifact ถามก่อน:
1. คนอ่านเป็นใคร รู้อะไรอยู่แล้ว?
2. ต้องการ decide อะไรหลังอ่าน?
3. format ไหนเหมาะ (deck / memo / email)?

แล้วค่อย draft — จะได้ version ที่ตรงตั้งแต่รอบแรก
