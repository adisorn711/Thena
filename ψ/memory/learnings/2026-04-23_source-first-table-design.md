# Source-First Table Design

**Date**: 2026-04-23
**Source**: Session — Customer Tag PPT source summary slides

## Lesson

เวลาออกแบบ "summary slide" หรือ "overview table" ให้เรียง data ตาม **insight ที่อยากสื่อ** ไม่ใช่ตาม structure ของ data เดิม (เช่น อย่าเรียงตาม category หรือตาม alphabetical order โดยไม่คิด)

## Pattern

```
❌ Structure-first ordering
เรียงตาม Food → Travel → Shopping... (ตาม category เดิม)
→ ผู้ดูต้องไล่อ่านเพื่อหา Overlap/New เอง

✅ Insight-first ordering
Overlap → New → Existing
→ เล่าเรื่อง "ค่าที่ได้จาก enrichment work" ก่อน แล้วค่อยแสดง baseline
→ stakeholder เห็น ROI ของ enrichment ทันที
```

## Why This Matters

สำหรับ stakeholder presentation, การเรียงลำดับข้อมูลคือการ **frame the story**:
- Overlap ขึ้นก่อน = "งานทีมที่คิดมา 99 อัน มีประโยชน์ตรงกับ customer tag ที่เรามีอยู่"
- New ตามมา = "enrichment ยังช่วยเพิ่ม tag ใหม่ที่ไม่เคยมี"
- Existing ท้ายสุด = "baseline ที่ทำงานได้อยู่แล้วโดยไม่ต้องรอ"

## How to Apply

เวลาออกแบบ summary table ในสไลด์:
1. ถามก่อนว่า "message หลักที่อยากให้ผู้ดูรู้สึกหลังดูสไลด์นี้คืออะไร"
2. เรียงแถวตาม message นั้น
3. ถ้าต้องการ visual reinforcement ให้ใช้ color coding เสริม (ไม่ใช่แทน)
