# Layer Naming ต้องสะท้อน Reality ขององค์กร

การใช้คำที่ overpromise ใน deck สำหรับ CEO (เช่น "Real-time Triggers" ในองค์กรที่ยังไม่ automate ครบ) สร้าง expectation gap ที่เป็นอันตราย

**Why:** Tee ชี้ว่า "real-time" ใน Layer 2 ทำให้ดูเหมือน org พร้อม automate 100% ซึ่งไม่จริง → เปลี่ยนเป็น "Personalized Trigger" ที่ครอบทั้ง manual, scheduled, และ real-time

**How to apply:** เวลาตั้งชื่อ capability layer ให้เลือกคำที่ describe **intent + outcome** ไม่ใช่ mechanism — "Personalized" สื่อถึงความตรง, "Real-time" สื่อถึง speed ซึ่งอาจไม่ใช่ value จริง

# Workflow Diagram vs Capability Layer — คนละมุม

ใน deck เดียวกันต้องมีทั้งสองอย่าง แต่ต้องแยกให้ชัด:

- **Capability Layer** (5 Layers table) = WHAT we build — strategic, big picture
- **Workflow Diagram** = HOW it works — operational, step-by-step

**How to apply:** แต่ละ slide ควรเล่ามุมเดียว อย่า mix ทั้งสองมุมในสไลด์เดียว
