## Honest Feedback

**Friction 1 — สร้าง PPTX template โดยไม่ถาม layout preference ก่อน**
Tee บอก "white theme, good contrast, 4 หน้า" ซึ่งเป็น spec ที่ชัดพอสมควร แต่ design detail เช่น left navy panel vs full-width header, card grid vs bullet list ฉันตัดสินใจเองทั้งหมด ถ้า Tee มี brand guideline หรือ preference อื่น จะต้องทำใหม่ ควรถาม "มี reference deck ไหม?" ก่อน

**Friction 2 — EV tag list ยาวเกินไปก่อน constraint ชัด**
ตอนแรกที่เสนอ EV signals ฉัน list ออกมา 4-5 ตัวรวมทั้ง ev_showroom_visitor ซึ่งต้องรอ dealership data ทั้งๆ ที่ยังไม่รู้ว่า Tee มีข้อมูลนั้นไหม ถ้าถามก่อนว่า "dealership รับบัตรแล้วหรือยัง?" ก็จะไม่ต้องเสนอ tag ที่ impossible ก่อนแล้วค่อย backtrack

**Friction 3 — Dining section ใน PPTX ไม่แสดง definition ของ P1 tags**
ใน slide Dining ฉันแสดงแค่ชื่อ tag สำหรับ Priority 1 (10 ตัว) โดยไม่มี definition เพราะ space constraint แต่คนอ่าน deck คนอื่น (ไม่ใช่ Tee) อาจไม่รู้ว่า `brunch_person` คืออะไร ควรมี hover definition หรือ appendix slide ให้

## Lessons Learned

1. **Propensity ≠ Intent — ต้องแยกให้ชัดก่อนออกแบบ tag** เมื่อไม่มี direct behavioral signal (เช่น showroom visit) สิ่งที่ทำได้คือ profile-based propensity เท่านั้น ซึ่งมี precision ต่างกันมาก และ use case ต่างกัน ต้องบอก stakeholder ชัดว่าเรากำลังทำอะไร

2. **False positive คิดตั้งแต่ design ไม่ใช่ตอน validate** `fuel_declining` เป็นตัวอย่างดี — signal ดูดีในทฤษฎีแต่ confound กับ WFH/Grab ถ้าไม่กรองออก precision พัง ควรถาม "อะไรทำให้ false positive ได้?" ทุกครั้งที่เสนอ signal ใหม่

3. **Data availability ถามก่อนเสมอ — ก่อน design tag ใดๆ** Session นี้ confirm ได้ว่าถามแล้วได้คำตอบที่ unlock หรือ block หลาย tags ทันที แทนที่จะ design ไปก่อนแล้วค้นพีทีหลังว่าทำไม่ได้
