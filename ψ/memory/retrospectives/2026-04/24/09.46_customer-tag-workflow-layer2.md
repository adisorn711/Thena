## Honest Feedback

**1. Draft-before-commit ทำงานดีมาก**
การที่ draft workflow ให้ Tee review ก่อนเขียนจริงช่วยได้มาก ไม่ต้อง rewrite หลัง commit ซึ่งเป็น pattern ที่ควรทำต่อไป โดยเฉพาะ content ที่มี judgment call สูง

**2. Clarification ก่อน edit สำคัญมาก**
ถ้าไม่ถามก่อนว่า "workflow" หมายถึงอะไร อาจได้ diagram ที่ไม่ตรง — การถาม 2 คำถามก่อน (แก้ที่ไหน, ระดับไหน) ช่วยให้ผลลัพธ์ตรงขึ้นมาก

**3. Layer 2 name change เป็น conceptual shift ไม่ใช่แค่ rename**
การเปลี่ยนจาก "Real-time" → "Personalized" มันเปลี่ยน definition ของ Layer 2 ด้วย ควร check ว่ามีที่อื่นใน deck หรือ memo ที่ยังใช้ "Real-time Triggers" อยู่อีกไหม เพื่อ consistency

## Lessons Learned

**Layer naming ใน deck สำหรับ CEO ต้องสะท้อน reality ขององค์กรปัจจุบัน**
การใช้คำที่ overpromise (เช่น "real-time") เมื่อ infra ยังไม่พร้อม สร้าง expectation gap ที่เป็นอันตราย ควรเลือกคำที่ describe intent และ outcome ไม่ใช่ mechanism

**Workflow diagram และ Capability Layer เล่าคนละมุม ต้องมีทั้งคู่ใน deck**
- Capability Layer = WHAT we build (strategic)
- Workflow diagram = HOW it works (operational)
ต้องชัดว่าแต่ละ slide เล่ามุมไหน ไม่ mix กัน
