## Honest Feedback

**1. Arrow ใน automation slide อาจไม่ render สวยใน pptx**
วิธีที่ใช้ทำ arrow คือ shape triangle + rectangle ซ้อนกัน ไม่ได้ใช้ connector ของ pptx จริงๆ อาจดูไม่ clean เท่าที่ควร ควรหา alternative เช่น ใช้ line connector หรือ unicode arrow character แทน

**2. Key message บาง section อาจยาวเกินไปสำหรับ slide**
โดยเฉพาะ section 01 กับ 02 — ข้อความยาว 2 บรรทัด อาจทำให้ card ดูแน่นเกิน ควรทดสอบดูว่า font 11pt ใน Inches น้อยๆ render ได้ readable ไหม

**3. ยังไม่ได้ตอบโจทย์ automation ครบ**
วันนี้ออกแบบภาพใหญ่ของ automation แต่ยังไม่ได้ลงรายละเอียด: offer_tag_profile schema, rule configuration UI, suppression logic — ยังเป็น parked topics ที่ต้องคุยต่อ

## Lessons Learned

**Double-tagged systems enable clean automation** — เมื่อ entity ทั้งสองฝั่ง (customer + offer) ใช้ taxonomy เดียวกัน matching logic จะเรียบง่ายมาก (JOIN on tag) และ campaign manager ไม่ต้องรู้ SQL เพียงแค่ระบุ tag + condition

**Greenfield = design for future friction, not fix current pain** — เมื่อไม่มี bottleneck ตอนนี้ การออกแบบ automation ต้องคิดล่วงหน้าว่า pain จะเกิดตรงไหนเมื่อ scale
