## Honest Feedback

**1. ควรถามเรื่อง use case ก่อนออกแบบ taxonomy**
ถาม use case แต่ Tee ตอบว่า "อยากใช้หมดเลย" — แล้วก็ proceed เลย ถ้าย้อนกลับไปควร probe ต่อว่า use case ไหน priority สูงสุด เพราะมันจะกำหนด tag definition และ threshold ที่ต่างกัน การที่ framework cover ทุก use case เท่ากัน อาจทำให้ทีม focus ไม่ได้ใน Phase 1

**2. PowerPoint สร้างโดยไม่ถามว่า Tee อยากได้แบบไหน**
ออกแบบ visual design เอง (dark theme, gold accent) โดยไม่ได้ถาม preference แม้ผลออกมาดูดี แต่ถ้า brand ของ org เป็นสีอื่น หรือ Tee มีรูปแบบที่ต้องการ ก็จะต้องทำใหม่ ควรถามก่อน

**3. ยังไม่ได้ link กลับไปที่ Spend Recovery Engine**
Memory บอกว่ามี active project อยู่ แต่ session นี้ไม่ได้ถามเลยว่า D-Tag จะ connect กับ Spend Recovery ยังไง อาจเป็น complementary projects หรืออาจ replace กัน — ยังไม่รู้ ควร surface ขึ้นมาในครั้งหน้า

## Lessons Learned

1. **Data foundation first** — รู้ว่าทีมมี MCC + Merchant Name ทำให้ออกแบบ roadmap ที่ actionable ได้ทันที ไม่ใช่แค่ ideal state
2. **Tag ≠ Segment** — ความแตกต่างหลักของ D-Tag คือ multi-dimensional, dynamic, confidence-scored — ไม่ใช่แค่ "segment ที่ละเอียดขึ้น"
3. **CEO mandate ต้องการ quick win** — Phase 1 ที่ build จาก existing data สำคัญมากในบริบทนี้ อย่า over-engineer ตั้งแต่ต้น
