## Honest Feedback

**1. Tag library ใหญ่เกินไปสำหรับ session เดียว**
151 tags ใน 35 นาที เป็นงานที่ dense มาก บางส่วนที่ทำเร็วอาจไม่ได้รับ scrutiny เพียงพอ เช่น life stage tags ที่ infer จาก behavior ควรมีการ validate logic มากกว่านี้ก่อนแชร์ทีม ควรแนะนำ Tee ให้ทีม review definition ก่อน implement ไม่ใช่ treat ว่า final

**2. ไม่ได้ถาม business context ก่อนทำ library**
ทีมอยู่ใน sector credit card ไทย ควรถามก่อนว่า merchant ในพอร์ตส่วนใหญ่เป็นแบบไหน — ถ้า top merchants คือ convenience store + fuel ก็ควร prioritize tags ในมิตินั้นก่อน แทนที่จะสร้าง library กว้างๆ ที่ครอบคลุมทุก segment เท่ากัน

**3. Aggregation logic ค้างอยู่**
Tee ขอ aggregation logic ตั้งแต่ต้น แต่ session จบโดยยังไม่ได้ทำ ควรตั้ง expectation ก่อนว่า tag library จะใช้เวลาเท่าไร และ aggregation จะต้องรอ session ถัดไป ไม่ควรปล่อยให้ค้างโดยไม่บอก

## Lessons Learned

1. **Merchant Tags คือ enrichment signal ไม่ใช่ Customer Tag** — สองสิ่งนี้อยู่ต่าง layer และต้องสื่อสารให้ชัดกับทีมก่อน implement
2. **78% ทำได้โดยไม่รอ Enrichment** — การ classify tag ตาม data dependency ช่วย unlock quick win และ prioritize roadmap ได้จริง
3. **Tag definition ต้องมี signal ไม่ใช่แค่ label** — "foodie" ไม่มีประโยชน์ถ้าไม่รู้ว่าดูจาก MCC อะไร ที่ threshold ไหน lookback นานแค่ไหน
