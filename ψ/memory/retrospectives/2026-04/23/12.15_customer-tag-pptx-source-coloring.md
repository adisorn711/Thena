## Honest Feedback

**1. CATEGORIES data ไม่ sync กับ markdown 100%**
ตอนเพิ่ม Entertainment count จาก 25 → 29 มีความสับสนเล็กน้อยว่า `tattoo_enthusiast` ควรอยู่ใน Entertainment หรือ Life Stage — markdown บอก Life Stage แต่ใน append_tags_pptx.py ถูกเพิ่มใน Entertainment เพราะ copy-paste context นี้คือ technical debt ที่ยังค้างอยู่ category count ที่แสดงใน slide header อาจ off ไปหนึ่ง ถ้า Tee สังเกตเห็น ควร fix ตอนนั้น

**2. ENRICHMENT_MAP มี hardcode ที่ fragile**
Map 80+ entries ถูก hard-code ใน `.py` file แต่ source of truth คือ `customer_tag_library_v2_expansion.md` ถ้า library เปลี่ยน ต้องอัพ 2 ที่ ในอุดมคติควร parse markdown หรือมี single source เช่น YAML/JSON แต่ตอนนี้ยังโอรับ scope

**3. Session ขาด validation step**
หลัง run scripts ผมแค่ดู output slide count (36) แต่ไม่ได้ open file จริงเพื่อ verify visual ว่า legend ตำแหน่งดี, badge ไม่ overflow, enrichment text ไม่ตัดขาดกลางคำ Tee ต้องเป็นคนไป verify เอง ซึ่งอาจเจอ visual bug ที่ผมพลาด

## Lessons Learned

**Source-first table design**: เวลาออกแบบ "summary slide" ให้เรียง data ตาม insight ที่อยากสื่อ ไม่ใช่ตาม structure ของ data เดิม — Overlap→New→Existing เรียงแบบนี้เพราะมันเล่าเรื่อง "ค่าที่ได้จาก enrichment" ก่อน แล้วค่อยแสดง baseline
