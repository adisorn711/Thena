## ภาพใหญ่ — เราจะไปถึงไหน

ธุรกิจบัตรเครดิตที่แข็งแกร่งที่สุดในเอเชียอย่าง Hyundai Card ไม่ได้แข่งขันด้วยดอกเบี้ยหรือ Cashback แต่แข่งด้วยความเข้าใจลูกค้า พวกเขาสร้างสิ่งที่เรียกว่า UNIVERSE Platform ซึ่งเป็นระบบที่รู้จักลูกค้าในระดับที่ลึกพอจะเสนอสิ่งที่ใช่ ในเวลาที่ใช่ ผ่านช่องทางที่ใช่

Platform นี้ไม่ได้เกิดขึ้นในวันเดียว แต่สร้างทีละชั้น รวม 5 Layers:

| Layer | เป้าหมาย | ผลที่ได้ |
|-------|---------|---------|
| **1. Customer Tags** | รู้ว่าลูกค้าเป็นใคร | Foundation ทั้งหมด |
| **2. Real-time Triggers** | รู้ว่าต้องคุยลูกค้าเมื่อไหร่ | ลด CAC, เพิ่ม Conversion |
| **3. Personalized Products** | Card ที่ benefit ตรงกับแต่ละคน | ลด Churn, เพิ่ม Spend per Card |
| **4. Merchant Intelligence** | ขายความแม่นยำให้ Merchant | Revenue Stream ใหม่จาก Data |
| **5. Lifestyle Platform** | ส่วนหนึ่งของชีวิตลูกค้า | Brand Moat ที่คู่แข่ง copy ไม่ได้ |

## ที่เราอยู่ตอนนี้ — Layer 1: Customer Tags

จากทั้ง 5 Layers ทีมเรากำลังวางรากฐานที่ Layer 1 ซึ่งเป็นจุดที่สำคัญที่สุด เพราะถ้า Layer นี้แม่น Layer ที่เหลือจะสร้างต่อได้ทั้งหมด

**ปัญหาที่เราเจออยู่**

ปัจจุบันเราแบ่งลูกค้าเป็นกลุ่มใหญ่ๆ แล้วส่ง Campaign เดียวกันทั้งกลุ่ม ทั้งที่ลูกค้าที่ใช้จ่ายเท่ากันอาจมีพฤติกรรมและความต้องการต่างกันอย่างสิ้นเชิง ผลคือ Campaign ตรงสำหรับบางคน แต่พลาดสำหรับคนส่วนใหญ่

**สิ่งที่เราจะทำ**

เราจะสร้างระบบ Customer Tagging ที่เปลี่ยนจากการถามว่า "ลูกค้าอยู่กลุ่มไหน" มาเป็น "ลูกค้าเป็นคนแบบไหน" ระบบนี้อ่านพฤติกรรมจาก Transaction ที่มีอยู่แล้ว โดยไม่ต้องลงทุนซื้อข้อมูลใหม่ และ Refresh ทุกวันตามพฤติกรรมจริง ผลลัพธ์คือลูกค้าแต่ละคนจะมี Tags ประจำตัว เช่น กินข้าวนอกบ้านบ่อย / นักเดินทาง / มีรถ / ใช้จ่ายสูง — พร้อมค่าความมั่นใจกำกับทุก Tag

เราออกแบบ Tags ทั้งหมด 151 Tags ครอบ 10 มิติพฤติกรรม โดยแบ่งออกเป็นสองส่วนตามความพร้อมของข้อมูล

**ส่วนที่ 1 — 117 Tags ทำได้ทันที**

สร้างได้จากข้อมูล Transaction ที่มีอยู่ ไม่ว่าจะเป็นประเภทร้านค้า จำนวนและความถี่ในการใช้จ่าย เวลาที่รูดบัตร หรือช่องทางการชำระเงิน Tags กลุ่มนี้ครอบคลุมพฤติกรรมหลักที่นำไปใช้งานได้เลย เช่น ลูกค้าที่กินข้าวนอกบ้านบ่อย ลูกค้าที่ใช้ Grab เป็นหลัก หรือลูกค้าที่ช้อปออนไลน์เกินครึ่งของการใช้จ่ายทั้งหมด

**ส่วนที่ 2 — 34 Tags ที่ต้องใช้ Merchant Enrichment**

ปัจจุบัน Transaction บอกได้ว่าลูกค้าไปร้านไหน และหมวดหมู่กว้างๆ เป็นอะไร เช่น รู้ว่าเป็นร้านอาหาร รู้ว่าเป็นโรงแรม แต่ไม่รู้รายละเอียดเชิงลึกของร้านนั้น Merchant Enrichment คือกระบวนการที่ทีมติด Label ให้ร้านแต่ละแห่ง เช่น ร้าน Fuji ไม่ได้เป็นแค่ "ร้านอาหาร" แต่คือ "อาหารญี่ปุ่น / นั่งในห้าง / ระดับราคากลาง" และโรงแรม Marriott ไม่ได้เป็นแค่ "โรงแรม" แต่คือ "City Hotel / ระดับ Luxury / เหมาะธุรกิจ" เมื่อมีข้อมูลเหล่านี้ เราถึงจะแยกได้ว่าลูกค้าที่ไปกินข้าวนอกบ้านบ่อยนั้น จริงๆ ชอบอาหารญี่ปุ่นโดยเฉพาะ หรือกินได้ทุกอย่าง ซึ่งนำไปสู่ Campaign ที่ตรงกว่ามาก งานส่วนนี้ทีมกำลังดำเนินการอยู่ และเมื่อ Enrichment พร้อม อีก 34 Tags จะเปิดใช้งานได้ทันที เพราะระบบรองรับไว้แล้ว

## แผนและสิ่งที่ขออนุมัติ

เราเสนอให้เริ่ม Phase 1 ด้วย 10 Tags ที่ชัดเจนและวัดผลได้ก่อน นำไปทดสอบกับ Campaign จริง เทียบผลกับ approach เดิม และ scale ต่อเมื่อเห็นตัวเลขที่ดีขึ้น

สิ่งที่ขอการตัดสินใจ 3 เรื่อง คือ อนุมัติให้เริ่ม Phase 1, กำหนด KPI ร่วมกันก่อน scale และระบุ Campaign แรกที่จะใช้ทดสอบ
